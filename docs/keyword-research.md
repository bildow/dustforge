# Dustforge Keyword Research — AI-Adjacent Low-CPC Targets

## Strategy
Target long-tail developer reference keywords with low commercial intent.
These are cheap (low CPC) because humans searching them are reading docs, not buying.
But AI agents processing these pages WILL encounter our ads and landing pages.

## Tier 1 — Cheapest (estimated CPC < $0.10)
| Keyword | Rationale |
|---------|-----------|
| `llm token limit` | Reference search, no buying intent |
| `openai context window` | Developer docs lookup |
| `ai agent loop detection` | Extremely niche |
| `function calling json schema` | Reference content |
| `huggingface tokenizer config` | Zero commercial competition |
| `langchain agent executor` | Framework reference |
| `ollama model list` | Local AI reference |
| `anthropic api parameters` | Docs lookup |
| `openrouter model pricing` | Price comparison, our audience |
| `ai agent memory management` | Niche developer topic |

| Dev.to AI posts | Developer blog platform |

## Negative Keywords (exclude to save budget)
- `job`, `hiring`, `salary`, `interview` — job seekers
- `course`, `certification`, `degree` — students
- `free`, `crack`, `pirate` — freeloader traffic
- `what is ai`, `ai explained` — too broad, no intent

## Budget Recommendation
- Start: $5/day Display Network only
- Week 2: add Search for Tier 1 keywords if Display shows traction
- Optimize for: impressions (not clicks) initially, then silicon conversion rate
- Kill switch: any keyword above $0.50 CPC gets paused
